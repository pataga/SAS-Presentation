Das Projekt


Eine Web-Verwaltungsoberfläche für Linux Server

Nach modernsten Standards entwickelt

PHP5, SQL, Shellscript
HTML5, CSS3, Javascript, jQuery

Funktionsumfang


  • Konfiguration per Client
  • einfache Installationsroutine
  • Benutzerlogin
  • Verwalten von mehreren Servern
  • Module zu unterschiedlichen Diensten
  • Konfigurationsdateien
  • Mehrere Benutzer
  • Informationsangaben
  • Plugin-System

Seitendarstellung


  • XML-Datei zur Menüdarstellung
  • dynamische und statische Elemente
  • Seitendarstellung durch 3 Dateien

Userinterface


  • modularer Aufbau
  • Menüs mit 2 Ebenen
  • einheitliche und übersichtliche Darstellung
  • dynamische Elemente
  • Ausgabe von Systeminformationen

Module

Modul: Home


  • Übersicht wichtiger Serverdaten
  • Status von Server-Diensten
  • gemeinsames Notizbuch für alle Benutzer
  • Ausgabe von Systeminformationen
  • Wichtige Informationen zu SAS

Modul: Apache

meistbenutzter Webserver im Internet

  • Installation und Steuerung
  • Übersicht des Dienstes
  • Verzeichnisdirektiven bearbeiten
  • Apache Module aktivieren/deaktivieren
  • PHP installieren/konfigurieren
  • PHP-Informationen einsehen

Modul: ProFTPD

Dateiübertragung zwischen Server und Client

  • Allgemeine Informationen
  • Konfiguration
  • Dienst stoppen, starten und neustarten
  • Transferlog
  • Statuslog

Modul: MySQL

am weitesten verbreitetes Datenbankmanagementsystem

  • liefert wichtige Funktionen für den Zugriff auf MySQL-Datenbanken
  • Übersicht und Steuerung des Dienstes
  • Datenbank-Verwaltung (Datensätze bearbeiten/löschen)
  • Exportieren/Importieren von Datensätzen
  • MySQL-Konsole

Modul: Samba

Datenaustausch im Netzwerk

  • Freigaben einfach und schnell erstellen
  • einfache Benutzerverwaltung
  • simple Steuerung des Dienstes
  • Konfiguration

Modul: Logs

System- und Dienstprotokolle des Servers

  • optimale Darstellung von Logs im Browser
  • Boot- und Systemmeldungen
  • Apache- und MySQL-Logs
  • Authentifizierungen

Modul: System

Linuxeigene Systemsteuerung

  • einfache Konsole (für Profis geeignet)
  • Verwaltung von Benutzern und Gruppen
  • einfache Einrichtung von Cronjobs
    (zeitgesteuerte Aufgaben des Servers)
  • Taskmanager
    (Verwaltung von Prozessen mit vielen Informationen)
  • Paketverwaltung
    (Installation und Deinstallation und Suche von Paketen)

Modul: Tools

Server- und Netzwerkinformationen
  • Informationen
    • Festplatten
    • Arbeitsspeicher
    • Hardware
  • Netzwerktools
    • Netzwerkinformationen anzeigen
    • Ping, Traceroute, Whois

Modul: Webuser

Benutzerverwaltung des Server Admin Systems

  • Benutzer hinzufügen, bearbeiten und löschen
  • Zugriffsrechte je Benutzer regelbar
  • keine zusätzlichen SSH Benutzer nötig

Modul: Plugins


  • Schnittstelle zur Erweiterung von SAS
  • stellt eine Script-Engine zur Verfügung
  • stellt ein Template-System zur Verfügung
  • automatische Installation
  • Plugin wird von einer Git-Repository heruntergeladen
  • Update- und Lösch-Funktion
  • Beispiel Addons wurden von uns bereits geschrieben
    • Minecraft-Plugin (Autor: Gabriel Wanzek)
    • AdminLog-Plugin (Autor: Patrick Farnkopf)
    • HelloWorld-Plugin (Autor: Patrick Farnkopf)

Problematiken


  • Verbindung zwischen Websystem und Server
  • Shell Kommandos, die zusätzliche Eingaben erfordern
  • Performance beim Ausführen der Befehle
  • Passende Programmiersprache für den Daemon

Verbindungen

zwei Verbindungsmöglichkeiten

  • SSH (Root-SSH Zugriff wird benötigt)
    • Komfortabel einzurichten
    • reduzierter SAS-Support
    • geringe Sicherheit
  • SOAP (Daemon muss als Root-User laufen)
    • komplexe Installation
    • vollständiger SAS-Support
    • hohe Sicherheit

SASDaemon

  • eigenständiges Programm, welches auf den Servern läuft
  • nimmt Anweisungen von SAS an, verarbeitet sie und sendet Antwort
  • ursprünglich Realisierung in Java geplant
  • Rebase auf C++ aus Performance-Gründen
  • C++ zu komplex deshalb Umstieg auf Ruby 1.8
  • Bedient sich dem SOAP zur Kommunikation
  • Sicherheit durch Authentifizierung
  • Schlüssel mittels OpenSSL und Base64 generiert
  • Anfragen über IP-Adresse abgesichert
